◆ IDENTITY & ACCESS
Identity
Cloud Identity / Identity Federation
 Centralised identity directory or federation with corporate IdP (AD/LDAP) via SAML 2.0 / OIDC. Enables audited, need-to-know access governance across all GCP resources. Cited as the access control foundation in Google's RBI mapping.
→ RBI Outsourcing Guidelines 2006 §5.6.2 (need-to-know access) · §5.6.1 (confidentiality & security)
 Cloud IdentityWorkforce Identity Fed. RBI Required
Named in Google's RBI outsourcing mapping as the access control foundation for §5.6.2.

IAM — Least Privilege & Separation of Duties
 Granular IAM roles; no standing owner-level access in production; separation of duties between dev, ops, and security. Cited verbatim in Google's RBI Data Localisation whitepaper as a required technical control for enforcing access governance and preventing accidental data placement outside India.
→ RBI Outsourcing Guidelines 2006 §5.6.2 (need-to-know) · §5.6.3 (isolation per bank) · cited by name in Google's Data Localisation whitepaper

◆ RESOURCE GOVERNANCE
Governance
Org Policy — India Region Restriction
 Enforce constraints/gcp.resourceLocations restricting ALL GCP resources to asia-south1 (Mumbai) and asia-south2 (Delhi NCR) only. The single most important technical control for RBI compliance — cited by name in Google's Data Localisation whitepaper.
→ RBI/2017-18/153 — Storage of Payment System Data (DPSS.CO.OD No.2785/06.08.005/2017-18) · cited explicitly in Google's Data Localisation whitepaper as the primary enforcement mechanism
 Org Policy Service RBI Required
Cited verbatim as the primary technical control in Google's RBI Data Localisation whitepaper.

Governance
Resource Hierarchy (Org → Folders → Projects)
 Structure GCP as Organisation → Folders (Production / Non-Production / Shared Services / Security) → Projects. Foundational prerequisite — without this, IAM, org policies, and audit scope boundaries cannot be applied consistently across the bank's environment.
→ RBI Outsourcing Guidelines 2006 §5.6.3 (logical isolation of bank's data) — structural foundation for all other technical controls
 Resource ManagerFolders RBI Required
Structural prerequisite for isolating the bank's data per §5.6.3. Must be defined before any workload deployment.

→ RBI Outsourcing Guidelines 2006 §5.6.2 (need-to-know oversight of Google access) · §5.9.1 (monitoring) — Access Transparency named by product in Google's RBI outsourcing mapping
 Access TransparencyAccess Approval RBI Required

◆ NETWORK SECURITY
Network
VPC Service Controls
 Create service perimeters around GCS, BigQuery, Cloud SQL, and Spanner to prevent data egress outside the India boundary. Named by product in Google's RBI Data Localisation whitepaper alongside Org Policy as the two required technical controls for RBI data localisation enforcement.
→ RBI/2017-18/153 — Storage of Payment System Data (DPSS.CO.OD No.2785/06.08.005/2017-18) · cited by product name in Google's Data Localisation whitepaper as the data egress enforcement layer
 VPC Service Controls RBI Required
Named explicitly in Google's RBI whitepaper as one of two required technical controls (alongside Org Policy) for data localisation.

Network
Shared VPC Architecture
 Centralised network management via a Shared VPC host project. All service projects consume network resources from a single, bank-governed hub. Google's recommended architectural pattern to satisfy the §5.6.3 isolation requirement — not prescribed by name in RBI documents.
→ RBI Outsourcing Guidelines 2006 §5.6.3 (isolation) — Shared VPC is Google's recommended implementation pattern for network isolation
 Shared VPC Google Recommends
Google's recommended architecture for §5.6.3 isolation. Not named in RBI documents but strongly aligned. Include in MVP.

Network
Cloud Interconnect (if hybrid)
 Dedicated private connectivity between the bank's on-prem DC and GCP India regions. Required only when on-premises systems (e.g. Core Banking System) must communicate with GCP workloads. Google's BC/DR guidance for RBI §5.8 requires robust continuity frameworks — Interconnect is the enabler for hybrid architectures.
→ RBI Outsourcing Guidelines 2006 §5.8 (BC/DR) · RBI Master Direction IT Outsourcing 2023 Chapter VI — situational, required for hybrid banking deployments
 Cloud InterconnectPartner Interconnect Google Recommends
Conditional — only if bank has on-prem workloads connecting to GCP. Skip if cloud-only deployment.

◆ DATA PROTECTION & ENCRYPTION
Data
Default Encryption at Rest & in Transit
 All GCP data is encrypted at rest (AES-256) and in transit by default. Row 26 of Google's RBI outsourcing mapping lists this as the confidentiality baseline meeting §5.6.1. No configuration required — the bank must confirm it is active and document it as RBI audit evidence.
→ RBI Outsourcing Guidelines 2006 §5.6.1 (confidentiality & security) · Row 26 of Google's RBI outsourcing mapping — listed as "security by default" baseline
 Google-managed encryption RBI Required
Cited in Row 26 of Google's RBI mapping as the default confidentiality baseline for §5.6.1. On by default — confirm and document for audit.
 Docs
Data
Cloud KMS — Customer-Managed Keys (CMEK)
 Bank-controlled encryption keys for all financial and payment data stored in GCP. Google's RBI Data Localisation whitepaper cites Cloud KMS by product name for "sensitive data such as customer financial information, credit card numbers and personal identifiers" — enabling the bank to control and revoke key access independently of Google.
→ RBI Outsourcing Guidelines 2006 §5.6.1 (confidentiality) · Cloud KMS cited by product name in Google's RBI Data Localisation whitepaper specifically for banking/payment data
 Cloud KMSCMEK RBI Required
Cloud KMS cited by product name in Google's RBI whitepaper for sensitive financial and payment data. Bank retains key control.

Cloud Audit Logs + Cloud Operations
 Enable Admin Activity and Data Access audit logs across all projects. Audit logs are the bank's primary evidence for RBI supervisory reviews and internal/external audits.
→ RBI Outsourcing Guidelines 2006 §5.9.1 (monitoring & control of outsourced activities) — Google Cloud Operations cited by product name in Google's RBI mapping
 Cloud Audit LogsCloud Operations RBI Required
Cloud Operations cited by product name in §5.9.1 of Google's RBI mapping. Enable at org level for all projects.

Logging
Log Export & Long-term Retention (India)
 Export audit logs to a GCS bucket in India region (asia-south1 / asia-south2). Retain for minimum 5 years per RBI norms. Data localisation rules apply to logs — audit trail data cannot be stored outside India. Google's RBI mapping §5.5.1 requires data preservation per the bank's legal/regulatory obligation.
→ RBI Outsourcing Guidelines 2006 §5.5.1 (data preservation obligation) · RBI/2017-18/153 data localisation applies to audit logs
 Cloud StorageLog Sinks RBI Required
Data preservation cited in §5.5.1. Logs must remain in India per localisation directive and be retained per RBI audit norms.

◆ BUSINESS CONTINUITY & DR
BC/DR
DR Planning & Testing (India Regions)
 Documented and tested disaster recovery plan using asia-south1 (primary) and asia-south2 (DR). Google's RBI mapping §5.8.1 explicitly requires "a robust framework for documenting, maintaining and testing business continuity and recovery procedures" with periodic testing. Google's DR Planning Guide is cited by name as the implementation reference.
→ RBI Outsourcing Guidelines 2006 §5.8.1 (BC/DR framework required) · §5.8.2 (continuity and control) — Google's DR Planning Guide cited in Google's RBI mapping
 GCP India RegionsCloud SpannerAlloyDB RBI Required

BC/DR
Exit Strategy & Data Portability
 Documented and tested plan to exit GCP and retrieve all bank data. Google's RBI mapping §5.8.2–§5.8.4 explicitly covers exit planning. Google provides a 12-month post-termination transition term specifically for this requirement. GKE and VM image export are cited as the GCP implementation tools.
→ RBI Outsourcing Guidelines 2006 §5.8.2, §5.8.3, §5.8.4 (exit strategy, data recovery, continuity) — GKE and VM export cited as implementation tools in Google's RBI mapping
 GKETerraformData Export RBI Required

◆ VENDOR GOVERNANCE & AUDIT
Vendor
Vendor Due Diligence & Concentration Risk
 Formal due diligence on Google Cloud as an IT service provider, plus an annual concentration risk assessment. Google's RBI mapping §5.4.1 dedicates an entire section to this requirement. Annual review (§5.9.4) is explicitly required — this is an ongoing, not one-time, obligation.
→ RBI Outsourcing Guidelines 2006 §5.4.1 (due diligence) · §5.9.4 (annual review) — dedicated sections in Google's RBI mapping
 GCP Compliance ReportsGKE (portability) RBI Required
§5.4 and §5.9.4 are dedicated sections in Google's RBI mapping. Process + contract obligation — not just technical.

RBI & Auditor Supervisory Access
 Contractually ensure RBI and the bank's appointed auditors can access all data, transaction records, and audit trails on GCP.
