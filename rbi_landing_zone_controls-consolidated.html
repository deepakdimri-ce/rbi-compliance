◆ RBI CYBERSECURITY FRAMEWORK (2016) — ADDITIONAL REQUIRED CONTROLS
Cybersecurity
Cyber Security Policy & Board Governance
 A Board-approved Cyber Security Policy, distinct from the broader IT Policy / IS Security Policy, is mandatory. The policy must define the cyber security strategy addressing prevention, detection, response, and recovery. Senior management must confirm readiness to the RBI Cyber Security and IT Examination Cell.
→ RBI Cybersecurity Framework for Banks (June 2016) — Board-approved Cyber Security Policy requirement. GCP supports this through its security controls and compliance documentation.
 Org PolicySCC PremiumGCP Security Controls RBI Required
RBI Cybersecurity Framework mandates a distinct, Board-approved Cyber Security Policy. GCP provides the technical controls foundation; the bank owns the policy.

Cybersecurity
Security Operations Centre (SOC) — 24×7 Monitoring
 A Security Operations Centre must be set up at the earliest with continuous surveillance capability to monitor and manage cyber risks in real time. The SOC must be regularly updated on emerging threats including DDoS, ransomware, zero-day attacks, and targeted attacks.
→ RBI Cybersecurity Framework for Banks (June 2016) Annex 2 — SOC configuration and requirements. Chronicle SIEM/SOAR is Google's recommended platform for building a GCP-native SOC.

◆ MASTER DIRECTION — DIGITAL PAYMENT SECURITY CONTROLS (2021)
Digital Payments
Application Security — Secure SDLC & OWASP Testing
 Regulated Entities must implement application security controls for all digital payment products. Source code must be tested for vulnerabilities including OWASP / OWASP Mobile Top 10. If source code is not owned by the RE, the application developer must certify it is free of known vulnerabilities, malware, and covert channels. All internet banking and mobile payment applications must undergo security testing before deployment.
→ RBI Master Direction on Digital Payment Security Controls (2021) — Application security and OWASP requirements for internet banking and mobile payments. GCP supports via Binary Authorization and Artifact Registry vulnerability scanning.
 Binary AuthorizationArtifact RegistryWeb Security ScannerCloud Build (SAST/DAST) RBI Required
RBI mandates OWASP testing for all digital payment apps. Binary Authorization prevents deployment of unvetted containers; Web Security Scanner detects OWASP vulnerabilities.
 Docs
Digital Payments
Multi-Factor Authentication for Payment Transactions
 Additional Factor of Authentication (AFA / 2FA) is mandated by RBI for all payment transactions. OTP-based authentication, digital signatures, and Key-based Message Authentication Codes (KMAC) for payment or fund transfer transactions must be implemented. OTP time windows must not exceed 100 seconds on either side of server time.
→ RBI Master Direction on Digital Payment Security Controls (2021) — AFA/2FA requirements for all payment transactions and internet banking. Extends beyond general MFA to payment-specific authentication controls.
 Cloud Identity (MFA)reCAPTCHA EnterpriseFirebase App CheckApigee (API auth) RBI Required
RBI mandates AFA for all payment transactions. This is separate from the general MFA for GCP access — it specifically applies to payment transaction authentication in the lending app.

Digital Payments
Fraud Risk Management & Transaction Monitoring
 REs must implement robust fraud risk management including surveillance and monitoring of digital payment transactions, rule-based and ML-based fraud detection, and transaction limits at card, BIN, and RE level. Overseas transaction monitoring and setting of transaction limits is mandated. HSMs used for payment security must have logging enabled with tamper-proof logs.
→ RBI Master Direction on Digital Payment Security Controls (2021) — Fraud Risk Management and transaction monitoring requirements. GCP supports via Vertex AI real-time fraud scoring and Cloud Pub/Sub event streaming.
 Vertex AI (fraud/AML)Pub/SubDataflowBigQuery (analytics)Cloud HSM (payment HSM) RBI Required
RBI mandates fraud risk management for all digital payment products. Vertex AI enables real-time ML fraud scoring; Cloud HSM secures payment signing keys with tamper-evident logging.

Digital Payments
Network & Database Security for Payment Systems
 Network and database security must be comprehensively reviewed. Connections to networks and databases that are allowed for a specified period must be closed after use — persistent open connections are a known vulnerability. ATMs running unsupported operating systems are prohibited. Payment system infrastructure must be isolated from general corporate networks.
→ RBI Master Direction on Digital Payment Security Controls (2021) — Network and database security requirements. Also aligned with RBI Cybersecurity Framework network security requirements.
 VPC Service ControlsCloud SQL (IAM auth)Hierarchical FirewallsCloud ArmorPrivate Service Connect RBI Required
RBI mandates network isolation of payment systems. VPC Service Controls, Private Service Connect, and hierarchical firewalls together enforce the required separation.

◆ RBI DIGITAL LENDING DIRECTIONS 2025 — DLA PLATFORM CONTROLS
Digital Lending
Borrower Data — India-Only Storage & Consent Audit Trail
 All borrower data collected by the Digital Lending App must be stored exclusively in India. Data collection must be need-based with explicit borrower consent and an immutable audit trail. DLAs must not access device resources (contacts, call logs, files). The Regulated Entity must define clear data policies covering type, retention period, use restrictions, and destruction protocols.
→ RBI Digital Lending Directions 2025 — Data storage, consent, and privacy requirements for DLAs. Reinforces RBI/2017-18/153 data localisation for lending data. GCP enforces via Org Policy region lock + VPC Service Controls.
 Org Policy (region lock)VPC Service ControlsCloud Spanner (consent store)Cloud DLP RBI Required
RBI Digital Lending Directions 2025 mandate India-only storage of borrower data and an audit trail for all consent events. Org Policy + VPC SC enforce localisation; Cloud Spanner stores immutable consent records.
 RBI Doc
Digital Lending
DLA Technology Standards & Cybersecurity Compliance
 The Regulated Entity is responsible for ensuring the DLA — and any Lending Service Provider it engages — complies with all RBI technology standards and cybersecurity requirements. A comprehensive privacy policy compliant with applicable laws and RBI guidelines must be in place and published in the app. The RE cannot delegate this responsibility to its technology vendor.
→ RBI Digital Lending Directions 2025 — Technology standards and cybersecurity compliance for DLAs. Google is not an LSP — it is the cloud infrastructure provider. The RE owns end-to-end technology compliance. GCP supports via compliance certifications and SCC Premium.
