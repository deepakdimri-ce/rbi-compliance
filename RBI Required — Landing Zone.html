BC/DR
Exit Strategy & Data Portability
 GCP: Recognises that REs need to be able to exit GCP Services without undue disruption, without limiting compliance with regulatory requirements, and without any detriment to continuity and quality of service to customers. Upon request, Google will continue to provide the Services for 12 months beyond the expiry or termination of the contract. Google will enable the RE to access and export data throughout the contract duration and during the post-termination transition term in industry standard formats. Customer: Maintain a documented and tested exit plan. Use GKE (portable container workloads), VM image export, and Terraform state for data retrieval and workload portability.

Cybersecurity
Cyber Security Policy & Board Governance
 A Board-approved Cyber Security Policy, distinct from the broader IT Policy / IS Security Policy, is mandatory. The policy must define the cyber security strategy addressing prevention, detection, response, and recovery. Senior management must confirm readiness to the RBI Cyber Security and IT Examination Cell.

◆ DIGITAL PAYMENT SECURITY CONTROLS (2021) & DIGITAL LENDING — APPLICATION LAYER
Digital Payments
Application Security — Secure SDLC & OWASP Testing
 Regulated Entities must implement application security controls for all digital payment products. Source code must be tested for vulnerabilities including OWASP / OWASP Mobile Top 10. If source code is not owned by the RE, the application developer must certify it is free of known vulnerabilities, malware, and covert channels. All internet banking and mobile payment applications must undergo security testing before deployment.
→ RBI Master Direction on Digital Payment Security Controls (2021) · Binary Authorization

Digital Lending
Borrower Consent Audit Trail
 What: An immutable, queryable audit trail of all borrower consent events — consent given, consent withdrawn, purpose stated, timestamp, channel, and version of consent form presented. Required under RBI Digital Lending Directions 2025 and DPDPA 2023. GCP approach: Use Cloud Spanner or Firestore (asia-south1) as the consent store — append-only with no delete permission for application service accounts. Combine with Cloud Audit Logs to capture any attempted modification. Bucket Lock (WORM) on the GCS audit export protects the evidence chain.
